## 发现的问题

### 问题1: 题号过滤警告（解析版试卷）

**日志现象**:
```
[WARNING] [题号过滤] 丢弃题号1（非递增）
[WARNING] [题号过滤] 丢弃题号2（非递增）
[WARNING] [题号过滤] 丢弃题号2（非递增）
[WARNING] [题号过滤] 丢弃题号3（非递增）
```

**原因分析**:
解析版试卷包含：
1. 原题题号（1, 2, 3...）
2. 解析部分重复的题号（1, 2, 3...）
3. 答案部分可能再次出现题号

当前算法检测到题号非递增，自动丢弃重复题号。

**影响**:
- 轻微：最终识别到25题，置信度0.99，说明过滤逻辑正确
- 但产生了4条警告日志

**建议优化**:
1. **识别解析版格式**，区分原题区和解析区
2. **上下文判断**：检测"解析：" "答案：" 等关键词
3. **降级为DEBUG**：非错误情况，不应用WARNING级别

### 问题2: LibreOffice警告

**日志现象**:
```
Warning: failed to launch javaldx - java may not function correctly
```

**原因分析**:
LibreOffice尝试启动Java组件但失败（Java未安装或配置）。

**影响**:
- 微小：DOCX→PDF转换仍然成功
- 不影响功能，但产生警告日志

**建议优化**:
1. **安装JRE**: 在Dockerfile中添加 `default-jre-headless`
2. **或禁用Java**: LibreOffice添加 `--norestore --nofirststartwizard` 参数
3. **重定向stderr**: 捕获并过滤此类警告

### 问题3: 媒体匹配警告

**日志现象**:
```
[WARNING] [智能匹配] 还有未分配的元素: 1个表格, 0张图片，将分配给第一题
```

**原因分析**:
- 试卷包含无关元素（页眉表格、装饰性图片）
- 或题目未明确引用图表

**影响**:
- 中等：可能导致图表分配错误
- 当前采用"分配给第一题"的降级策略

**建议优化**:
1. **元素类型识别**：
   - 检测页眉/页脚区域的元素
   - 识别装饰性图片（小尺寸、纯色背景）
2. **关键词扩展**：
   - 除了"如图所示"，还检测"表中数据" "图示" "示意图"
3. **多题匹配**：
   - 一个图表可能对应多道题
   - 计算多个题目与图表的proximity分数

### 问题4: 大文件性能

**测试数据**:
- 25页文件耗时12.49秒
- 平均每页0.5秒

**瓶颈分析**:
```
[结构化提取] 完成，共25页  # 耗时最多
[题号检测] 完成，识别到25个题号
[题目分割] 完成，共25道题目
[媒体匹配] 完成
```

**建议优化**:
1. **并行处理页面**：
   - 使用多进程提取页面内容
   - 25页可分5个进程，理论提速4-5倍
2. **增量提取**：
   - 首次提取缓存结果
   - 人工校准后无需重新提取
3. **懒加载图片**：
   - 先提取文本和坐标
   - 图片按需提取

from multiprocessing import Pool

def extract_page(pdf_path, page_num):
    with pdfplumber.open(pdf_path) as pdf:
        page = pdf.pages[page_num]
        return extract_page_content(page)

def parallel_extract(pdf_path, total_pages):
    with Pool(processes=min(5, total_pages)) as pool:
        results = pool.starmap(extract_page,
            [(pdf_path, i) for i in range(total_pages)])
    return results
```

**预期效果**:

def detect_question_sections(structured_data):
    """检测试卷结构"""
    sections = {"questions": [], "answers": [], "explanations": []}

    for block in structured_data:
        if "参考答案" in block.text or "答案：" in block.text:
            sections["answers"].append(block)
        elif "解析：" in block.text or "详解：" in block.text:
            sections["explanations"].append(block)
        else:
            sections["questions"].append(block)

    return sections

def split_with_sections(pdf_path):

def is_decorative_element(element):
    """判断是否为装饰性元素"""
    # 页眉/页脚区域

def multi_question_image_match(questions, image):
    """一个图表可能对应多道题"""
    matches = []
    for q in questions:
        if has_image_reference(q.content):  # "如图所示"等
            score = calculate_proximity(q, image)
            if score > 0.3:

def clean_question_content(text):
    """清理题目文本"""
    # 移除页码
    text = re.sub(r'第\s*\d+\s*页\s*共\s*\d+\s*页', '', text)

    # 移除页眉信息
    text = re.sub(r'^\d{4}年.*?考试\n', '', text)

    # 标准化空白符
    text = re.sub(r'\s+', ' ', text)

    # 移除尾部的页码标记
    text = re.sub(r'\s*-\s*\d+\s*-\s*$', '', text)

    return text.strip()
```

#### 6. 置信度细化
**目标**: 提供更详细的置信度信息

**实现方案**:
```python
class ConfidenceDetail:
    overall: float  # 整体置信度
    pattern_match: float  # 题号匹配置信度
    boundary_clear: float  # 边界清晰度
    format_regular: float  # 格式规范性
    has_warnings: bool  # 是否有警告
    warnings: List[str]  # 警告列表

def calculate_detailed_confidence(questions):
    detail = ConfidenceDetail()

    # 题号匹配：标准模式(1.0) > 括号模式(0.9) > 其他(0.7)
    pattern_scores = [q.pattern_confidence for q in questions]
    detail.pattern_match = sum(pattern_scores) / len(pattern_scores)

    # 边界清晰度：检测题目间是否有明确分隔
    detail.boundary_clear = check_boundary_clarity(questions)

    # 格式规范性：题号递增、无跳号、无重复
    detail.format_regular = check_format_regularity(questions)

    # 综合置信度
    detail.overall = (
        detail.pattern_match * 0.4 +
        detail.boundary_clear * 0.3 +
        detail.format_regular * 0.3
    )

    return detail
```
