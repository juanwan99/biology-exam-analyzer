## 三、核心功能实现详解

### 3.1 高精度 Word 文档处理（最新实现）

#### 功能亮点

**问题背景**：
- Word 转 PDF 后 OCR 识别表格和文字准确率低
- 图表位置信息丢失

**解决方案**：混合策略
```python
# document_processor.py: extract_word_content()

1. python-docx 提取：
   - ✅ 文字段落（100% 准确）
   - ✅ 表格 → Markdown 格式（保留结构）
   - ✅ 嵌入图片（提取字节数据）
   - ✅ 保持元素顺序（段落-表格-图片）

2. LibreOffice 转 PDF：
   - ✅ 生成布局参考图（用于 Gemini 识别图表位置）

3. 合并发送给 Gemini：
   - 文字内容（Prompt 文本）
   - 布局图片（视觉参考）
```

#### 代码位置

```python
# backend/document_processor.py

extract_word_content(file_path)  # 主提取函数
    ├─ 遍历 doc.element.body (保持顺序)
    ├─ CT_P → 段落处理 + 内联图片提取
    ├─ CT_Tbl → 表格转 Markdown
    └─ 返回 {"text": "...", "images": [...], "elements": [...]}

_table_to_markdown(table)  # 表格转换
    └─ 输出格式: | 列1 | 列2 |
                  |-----|-----|
                  | 值1 | 值2 |

process_docx(file_path)  # Word 处理入口
    ├─ 调用 extract_word_content()
    ├─ LibreOffice 转 PDF
    ├─ PDF → 图片
    └─ 附加提取内容到 image.info['extracted_text']
```

### 3.3 Prompt 工程

#### split_prompt.txt（题目拆分）

```
请分析这份生物试卷，将其拆分为单独的题目。

**重要提示**：
1. 一道题可能跨多页，请识别完整题目后再分割
2. 题目编号（如"14."）是唯一标识，相同编号属于同一题
```

## 十、常见问题 FAQ

### Q1: Gemini API 调用失败怎么办？

**检查步骤**：
1. 验证 API Key 是否有效：`.env` 文件中的 `GEMINI_API_KEY`
2. 检查网络连通性：`curl https://generativelanguage.googleapis.com`
3. 查看后端日志：`docker-compose logs backend --tail=50`

### Q2: Word 文档中的表格没有被识别？

**可能原因**：
- Word 使用了文本框而非真实表格
- 表格格式不规范

**验证方法**：
```bash
docker exec biology_backend python3 -c "
from docx import Document
doc = Document('/app/uploads/xxx.docx')
print(f'表格数量: {len(doc.tables)}')
"
```
