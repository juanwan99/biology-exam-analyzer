### 重大问题修复记录

#### 问题1: 详细分析过短（30字符）
**症状**: 前端显示的详细分析只有一句话

**根本原因**: Docker容器内使用了旧版超严格prompt（限制50字符）

**诊断过程**:
```bash
# 1. 发现容器内prompt与本地不一致
docker exec biology_backend cat /app/prompts/analysis_prompt.txt

# 2. 发现docker-compose挂载的是 ./prompts 而非 ./backend/prompts
```

**解决方案**:
```bash
# 同步最新prompt到挂载目录
cp backend/prompts/analysis_prompt.txt prompts/analysis_prompt.txt
docker-compose restart backend
```

**关键发现**: 已配置卷挂载，修改prompt只需重启，无需重建容器（5秒 vs 3-5分钟）

#### 问题2: JSON解析失败 - "Unterminated string"
**症状**:
```
[ERROR] 题目7 API响应长度: 19887
[ERROR] 内容被截断！
JSONDecodeError: Unterminated string
```

**根本原因**: LLM生成19,887-21,138字符的超长分析，包含大量`\n`换行符，超出max_tokens=16384限制

**迭代修复过程**:

**版本1**: 添加字数限制和单行要求
```
详细分析（detailed_analysis）：
- 控制在200-300字
- 必须是单行文字，不得包含 \n 换行符
```
**结果**: 仍生成21,138字符 ❌

**版本2**: 添加表格说明（用户反馈）
```
- 不要在文字中重复描述表格内容（前端已渲染表格）
- 只需在分析中引用表格数据，如"根据表格数据可知..."
```
**结果**: 仍超长 ❌

**版本3（最终）**: 超严格禁令 + 负面案例
```
**【严格限制】detailed_analysis必须≤300字，禁止超长分析！**

- **【硬性限制】总字数必须≤300字，严禁超出！**
- **【禁止】详细推导、分步计算、逐代分析**
- **格式**：单行文字，用→、；分隔，禁止\n换行
- **正确示例**："本题考查伴性遗传概率计算。核心思路：推断基因型 → 计算配子比例 → 求纯合概率。选项：A错（未考虑雄性无纯合）；B对；C错（混淆遗传方式）；D错（计算有误）。关键是理解雄性无法纯合。"（98字）
- **错误示例**：不要写成"推断Ⅱ-2基因型为XalXa3或Xa2Xa3各占1/2，推断Ⅲ-1基因型..."这种详细推导
```
**结果**: 676-960字符，成功控制 ✅

**关键改进**:
1. 顶部用粗体警告引起注意
2. 更改措辞："详细解题步骤" → "简明解题思路"
3. 明确禁止具体行为（详细推导、分步计算）
4. 提供负面案例（更直观）
5. 示例从263字压缩到98字

### 已知问题

#### ⚠️ LLM推理准确性不足（复杂遗传题）
**案例**: 第7题（伴X遗传概率计算）

**题目**:
```
某动物家系的系谱图如图所示。a1、a2、a3、a4是位于X染色体上的等位基因，
Ⅰ-1基因型为XalXa2，Ⅰ-2基因型为Xa3Y，Ⅱ-1和Ⅱ-4基因型均为Xa4Y，
Ⅳ-1为纯合子的概率为（    ）
A. 3/64  B. 3/32  C. 1/8  D. 3/16
```

**LLM错误分析**:
1. 推导Ⅲ-1基因型时，错误给出 "1/4Xa3Xa4"
2. 忘记约束条件：父亲Ⅱ-4为Xa4Y，只能提供Xa4
3. 最终答案错误（选B，正确答案应为C）

**可能解决方案**（未实施）:
1. **升级模型**: Gemini 2.0 Flash → Gemini Pro（准确率更高，但成本5倍）
2. **双模型策略**: 简单题用Flash，复杂题（检测到"系谱图"、"概率"等关键词）自动切换Pro
3. **自我验证**: 让LLM检查自己的推理过程
4. **置信度评分**: 低置信度题目标记为"需人工审核"

## 🎯 项目二：教材-试题精准映射系统

### 背景需求
**用户原始需求**:
> "我需要的是这道题目、这个选项或者这句话是考查教材上具体的某一个知识点，来自教材具体某句话，教材某个课后习题的改编。"

**示例**:
- 题目选项A提到"细胞膜具有选择透过性"
- 系统应返回：
  - 来源：必修1 第62页 第3段 第2句
  - 原文："细胞膜是一种选择透过性膜，可以控制物质进出细胞。"
  - 类型：原文引用

### 技术验证过程

#### 第一步：PDF提取质量测试
**文件**: `test_pdf_extraction.py`

**测试对象**: 必修1《分子与细胞》（141页）

**测试结果**:
```
✅ 无乱码
✅ 字数正常（全书114,525字，平均812字/页）
✅ 中文字符占比84%
✅ 换行密度正常（5.4%）
```

**结论**: PDF为电子版，文本提取质量优秀，无需OCR

#### 第二步：向量检索准确性测试
**文件**: `test_vector_retrieval.py`

**索引范围**: 必修1前50页（80个片段，chunk_size=800字符）

**Embedding模型**: paraphrase-multilingual-MiniLM-L12-v2

**测试查询**:
| 查询 | 期望章节 | 期望关键词 | 命中率 | 实际情况 |
|------|---------|-----------|--------|---------|
| 细胞膜的结构和功能 | 第3/4章 | 细胞膜、磷脂双分子层 | **100%** ✅ | 内容在前50页 |
| 被动运输和主动运输的区别 | 第4章 | 被动运输、主动运输 | **0%** ❌ | 内容在第62页+ |
| ATP的结构和功能 | 第5章 | ATP、高能磷酸键 | **20%** ⚠️ | 仅目录有提及 |
| 光合作用的过程 | 第5章 | 光合作用、叶绿体 | **40%** ⚠️ | 部分匹配 |
| 细胞呼吸的类型 | 第5章 | 有氧呼吸、无氧呼吸 | **20%** ⚠️ | 部分匹配 |

**总体准确率**: **36%** ❌

**失败原因**: 只索引了前50页，但测试查询2-5的内容都在62页之后

**重要发现**: 向量检索对于"章节级匹配"有一定效果，但距离用户需求的"句子级精准映射"还有巨大差距

### 用户需求 vs 技术实现差距分析

#### 用户真实需求
1. **精度要求**: 句子级（非段落级、章节级）
2. **映射类型识别**:
   - 原文引用（100%一致）
   - 概念改写（80%相似）
   - 例题改编（识别原题号）
3. **输出格式**:
   ```
   选项A: "细胞膜具有选择透过性"
   → 来源：必修1 p.62 第3段第2句
   → 原文："细胞膜是一种选择透过性膜..."
   → 类型：原文引用
   ```

#### 当前技术能力
1. **精度**: 段落级（800字符chunk）
2. **映射类型**: 无法识别
3. **输出格式**: 只能返回相关段落，无法定位具体句子

#### 技术差距
| 需求维度 | 用户期望 | 当前能力 | 差距 |
|---------|---------|---------|------|
| 定位精度 | 句子级 | 段落级 | **巨大** |
| 映射类型识别 | 3种类型 | 无 | **巨大** |
| 改编题识别 | 课后习题追溯 | 无 | **巨大** |
| 准确率要求 | >95% | 36% | **巨大** |

### 可行性方案（未实施）

#### 方案A：纯向量检索（不推荐）
**步骤**:
1. 极细粒度切分（按句子，200字符/chunk）
2. 索引全部5本教材（141+179+120+95+112 = 647页）
3. 三层检索：粗召回 → 精排序 → LLM验证

**预估效果**: 准确率60-70%（仍不达标）

**时间成本**: 2-3天

**问题**: 无法识别改编题，无法区分引用类型

#### 方案B：人工标注 + LLM辅助（推荐）
**步骤**:
1. 人工标注必修1教材（按知识点结构化）
   - 每个概念定义标注页码、段落、句子
   - 每道课后习题录入题库
2. 构建知识点-教材映射JSON
3. LLM辅助：
   - 题目 → 知识点提取
   - 知识点 → 教材定位（查JSON）
   - 相似度计算（判断是原文/改写/改编）

**预估效果**: 准确率90-95% ✅

**时间成本**:
- 人工标注：5-8小时/本（共5本 = 25-40小时）
- 系统开发：3-5天

**优点**: 准确率高，可区分映射类型

**缺点**: 人工成本高，教材更新需重新标注

### 为何暂停该项目
1. **需求复杂度远超预期**: 从"章节匹配"升级到"句子定位"
2. **技术难度巨大**: 纯AI方案无法达到要求精度
3. **人工成本过高**: 需25-40小时标注
4. **用户转向其他方向**: 发现课标映射和难度评估更有价值

#### 方案C：难度量化评估（最推荐）✨
**内容**: 对题目难度进行多维度量化打分，生成可视化难度报告

**核心公式**:
```
难度系数 = (
    知识点复杂度 × 0.4 +
    认知层级 × 0.3 +
    信息提取难度 × 0.2 +
    步骤复杂度 × 0.1
)
```

**四个维度详解**:

1. **知识点复杂度（40%权重）**
   - 单知识点：0-3分
   - 跨章节（2-3个知识点）：4-6分
   - 跨模块（3+知识点）：7-10分

2. **认知层级（30%权重）** - 基于布鲁姆分类法
   - 记忆（识记）：0-2分
   - 理解（解释、举例）：3-5分
   - 应用（计算、预测）：6-8分
   - 分析/综合/评价：9-10分

3. **信息提取难度（20%权重）**
   - 纯文字：0-3分
   - 简单图表：4-6分
   - 复杂图表（系谱图、坐标图、多表格）：7-10分

4. **步骤复杂度（10%权重）**
   - 一步直达：0-3分
   - 2-3步推理：4-6分
   - 多步骤/多条件：7-10分

**示例分析** - 2025山东卷第7题:
```json
{
  "题目ID": 7,
  "综合难度": 8.2,
  "难度等级": "困难",
  "维度详情": {
    "知识点复杂度": 6.0,  // 跨章节（伴性遗传+概率计算）
    "认知层级": 9.0,       // 分析级（多代推导）
    "信息提取难度": 8.5,   // 系谱图+多个隐性条件
    "步骤复杂度": 9.0      // 4步推理（Ⅱ→Ⅲ→Ⅳ→概率）
  },
  "关键难点": [
    "系谱图解读",
    "多代基因型推导",
    "复杂概率计算",
    "隐性约束条件识别"
  ]
}
```

**可视化输出**:
1. **难度曲线图**: X轴=题号，Y轴=难度系数（折线图）
2. **维度雷达图**: 四个维度的雷达图（每道题）
3. **难度分布直方图**: 简单/中等/困难的题目数量分布
4. **试卷难度梯度评估**: "前紧后松" / "难度均衡" / "末题过难"

**实施路线**（4天）:
```
Day 1: 整理难度评估规则库
  - 构建知识点关键词库（模块、章节、知识点三级）
```

## 🎓 技术学习点

### Prompt工程经验
1. **字数限制无效问题**:
   - ❌ 错误："请控制在200-300字"（LLM会忽略）
   - ✅ 正确："**【硬性限制】总字数必须≤300字，严禁超出！**"（加粗+严厉语气）

2. **负面案例更有效**:
   - 仅提供正面示例：LLM理解不深
   - 同时提供正反示例："不要写成XXX"→ LLM明确边界

3. **格式控制技巧**:
   - ❌ "尽量避免换行" → LLM仍会使用`\n`
   - ✅ "禁止\n换行符，使用→、；分隔" → 强制替代方案

### Docker运维经验
1. **卷挂载 vs 文件复制**:
   - 卷挂载（`./prompts:/app/prompts`）：修改后重启即可，适合频繁迭代
   - COPY指令（Dockerfile）：修改后需重建，适合不变文件

2. **重启 vs 重建判断**:
   - 代码逻辑变化 → 必须重建
   - 数据/配置变化 → 只需重启

### RAG检索经验
1. **向量检索的局限性**:
   - 适合：模糊匹配、语义相似
   - 不适合：精确定位、句子级查找

2. **Chunk粒度权衡**:
   - 粗粒度（800字）：召回率高，精度低
   - 细粒度（200字）：精度高，但可能割裂语义

3. **准确率提升路径**:
   - 纯向量：36%
   - 向量+关键词混合：预估60-70%
   - 人工标注+AI辅助：90-95%
