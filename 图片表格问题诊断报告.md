## 1. 项目背景

### 1.1 项目概述
- **项目名称**: 生物试卷智能分析系统
- **核心功能**: 上传PDF/Word格式的生物试卷 → 拆分题目 → 逐题深度分析
- **技术栈**:
  - 后端: FastAPI + Python 3.12
  - AI引擎: Gemini 2.5 Flash (via OpenAI兼容接口)
  - 文档处理: pdf2image, pdfplumber, python-docx
- **当前进度**: 45%

### 1.2 预期功能流程
```
用户上传试卷 (PDF/DOCX)
    ↓
后端提取文字、表格、图片（结构化内容）
    ↓
智能匹配：将表格/图片分配给对应题目
    ↓
Gemini拆分题目
    ↓
Gemini逐题分析（使用题目+对应的表格/图片）
    ↓
返回前端展示（题目+分析+表格HTML+图片Base64）
```

## 2. 面临问题

### 2.1 问题描述
**用户反馈**: "图片和表格的问题依然没有解决"

### 2.2 已验证的事实（根据日志）
✅ **已成功的部分**:
1. Word文档成功提取了10个元素（1个表格 + 1张图片 + 8个段落）
2. PDF转换过程成功提取了4个元素
3. 智能匹配算法正常工作：
   - 题目7识别到"图"关键字，分配了图片 #1
   - 题目15识别到"表"关键字，分配了表格 #1
4. Gemini API调用成功，返回了题目分析结果

❌ **存在的缺陷**:
**提取的表格和图片没有传递给前端用户查看！**

### 2.3 核心问题定位
根据代码审查和日志分析，问题的根本原因是：

> **智能匹配算法将表格/图片添加到了 `question['structured_content']` 字段，
> 但API返回给前端时，这个字段被完全忽略，前端无法获取表格和图片数据！**

## 6. 风险评估

### 6.1 修复后的潜在风险
- **Base64图片数据过大**: 可能导致JSON响应体积暴增，需要考虑分页或压缩
- **Gemini Token消耗**: 传递更多高精度图片会增加API成本
- **API超时**: 大量图片传输可能导致请求超时

### 6.2 建议的优化方向
1. **图片压缩**: 提取的图片转为JPEG格式并压缩至合理大小（如100KB）
2. **CDN存储**: 将图片上传到对象存储（如阿里云OSS），API只返回URL
3. **表格优先**: 优先使用Markdown表格而非图片识别表格
