## 问题2: 规则拆分没有匹配到图片 ✅ 已修复

### 问题描述
题目7包含"如图所示"文字，但`auto_split`返回的`images`数组为空。

### 根本原因
原始IoU匹配算法只适用于图片**在题目bbox范围内**的情况。

生物试卷的典型布局：
```
7. 某动物家系的系谱图如图所示...  ← 题目文本
   A. 3/64  B. 3/32  C. 1/8  D. 3/16
   
   [系谱图图片]  ← 图片在题目文本下方，bbox不重叠
```

IoU（Intersection over Union）= 0，导致匹配失败。

### 解决方案
改进 `rule_splitter.py` 的 `_match_media_to_questions` 方法，增加**基于位置关系的匹配**：

```python
# 方法1：IoU匹配（图片在题目范围内）
iou = question.bbox.iou(img["bbox"])

# 方法2：垂直距离匹配（图片在题目下方）
if question.bbox.page == img["bbox"].page:
    if img["bbox"].y0 >= question.bbox.y0:  # 图片在题目下方
        vertical_distance = img["bbox"].y0 - question.bbox.y1
        x_overlap_ratio = ...  # 水平重叠度
        
        # 如果题目包含"如图所示"等关键词
        if any(kw in question.content for kw in ["如图所示", "如图", "下图", "见图"]):
            if vertical_distance < 200 and x_overlap_ratio > 0.3:
                proximity_score = 1.0 - (vertical_distance / 200) * (1.0 - x_overlap_ratio)
```

### 匹配策略
1. **IoU优先**：如果图片在题目bbox内（IoU > 0），优先使用IoU
2. **位置推理**：如果题目包含图表关键词，在下方200pt内查找图片
3. **水平对齐**：要求图片与题目有30%以上的水平重叠

## 问题3: 人工校准步骤去哪了？ ✅ 已确认正常

### 问题描述
用户测试时没有看到人工校准页面。

### 原因分析
用户选择了"⚡ 直接完整分析（v1.0经典模式）"，该模式**不包含人工校准步骤**。

### 工作流对比

| 步骤 | v1.0直接分析 | v2.0规则拆分+校准 |
|------|-------------|------------------|
| 1 | 上传文件 | 上传文件 |
| 2 | LLM自动拆分 | **规则引擎拆分** |
| 3 | - | **跳转到/correction页面** |
| 4 | - | **人工校准界面（编辑/合并/拆分/删除）** |
| 5 | - | **确认并继续** |
| 6 | 逐题分析 | 逐题分析 |
| 7 | 难度评估 | 难度评估 |
| 8 | 素养分析 | 素养分析 |
| 9 | 显示结果 | 显示结果 |

### 正确使用v2.0的步骤
1. 访问 http://localhost:3000
2. **选择"🎯 规则拆分 + 人工校准（推荐）"**
3. 上传PDF文件
4. 点击"开始拆分"（不是"开始分析"）
5. **等待跳转到 `/correction` 页面**
6. 在校准页面：
   - 查看拆分结果
   - 检查置信度标识（红/黄/绿）
   - 查看警告信息
   - 可选：编辑、合并、拆分、删除题目
7. 选择模式（快速/深度）
8. 点击"✅ 确认并继续分析"
9. 查看完整分析结果
