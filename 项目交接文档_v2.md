## 1. 项目概述

### 1.1 项目背景

本系统是为**高中生物试题审题模型**研发项目开发的智能分析工具，基于 Gemini 2.5 Flash AI模型，支持PDF和Word格式的试卷自动拆分与深度分析。

**研究目标：**
- 课标关键词智能映射（四大核心素养：生命观念、科学思维、探究实践、态度责任）
- 能力层级分解（七大能力维度）
- 学科逻辑分析（遗传题逻辑、实验设计检测）
- 三维评估报告生成

### 1.2 技术栈

**后端：**
- Python 3.11
- FastAPI 0.109.0
- OpenAI SDK 2.5.0（用于调用Gemini兼容API）
- python-docx 1.1.0（Word文档处理）
- pdf2image 1.17.0（PDF转图片）
- LibreOffice（Word转PDF）

**前端：**
- React 18
- Vite 5
- TailwindCSS 3
- Nginx（生产环境）

**AI模型：**
- Gemini 2.5 Flash（通过第三方OpenAI兼容接口调用）

**部署：**
- Docker + Docker Compose
- 云服务器（1核2G）

## 2. 最新更新（v2.0）

### 2.1 重大功能更新

#### ✅ 表格精确提取与HTML渲染
**问题：** Word表格被转换为Markdown后，前端无法正确渲染
**解决方案：**
1. 后端：`document_processor.py` 新增 `_markdown_table_to_html()` 方法
2. 后端：为每个表格元素同时保存 `markdown` 和 `html` 格式
3. 前端：`ResultDisplay.jsx` 使用 `dangerouslySetInnerHTML` 渲染HTML表格
4. 样式：`index.css` 添加 `.word-table` 样式（边框、表头背景、悬停效果）

**数据流：**
```
Word表格 → python-docx解析 → Markdown → HTML转换 → 前端渲染
```

#### ✅ 图片按原位置插入
**问题：** Word图片无法在题目中正确显示
**解决方案：**
1. 后端：提取图片时转换为 base64 编码
2. 后端：通过 `elements` 数组保持文档元素顺序（段落→表格→图片）
3. 后端：通过 `structured_content` 字段传递完整结构
4. 前端：`renderStructuredContent()` 按顺序渲染所有元素

**数据结构：**
```json
{
  "structured_content": [
    {"type": "paragraph", "content": "题目文本..."},
    {"type": "table", "html": "<table>...</table>"},
    {"type": "image", "base64": "data:image/jpeg;base64,..."}
  ]
}
```

#### ✅ API切换至第三方服务商
**问题：** Google官方免费API每天限制50次请求，已达配额上限
**解决方案：**

3. 模型升级：`gemini-2.0-flash-exp` → `gemini-2.5-flash`（⚠️ 不能降级）

**配置变更：**

## 4. 核心功能详解

### 4.1 高精度Word内容提取

**文件：** `backend/document_processor.py`

#### 方法：`extract_word_content(file_path) → Dict`

**功能：** 按顺序提取Word文档的所有元素（段落、表格、图片）

**返回数据结构：**
```python
{
    "text": "完整文本（包含Markdown表格）",
    "images": [
        {
            "index": 0,
            "data": bytes,           # 原始图片字节
            "base64": "iVBORw0KG...",  # base64编码
            "position": 1            # 在elements中的位置
        }
    ],
    "elements": [
        {"type": "paragraph", "content": "段落文本"},
        {"type": "table", "markdown": "| 表头 |\n|---|\n| 数据 |", "html": "<table>...</table>"},
        {"type": "image", "index": 0, "base64": "...", "caption": "图片说明"}
    ]
}
```

**关键技术：**
```python
# 遍历文档块级元素（保持顺序）
for block in doc.element.body:
    if isinstance(block, CT_P):          # 段落
        # 提取文字 + 内联图片
    elif isinstance(block, CT_Tbl):      # 表格
        # 转换为Markdown + HTML
```

#### 方法：`_table_to_markdown(table) → str`

**功能：** 将Word表格转换为Markdown格式

**示例：**
```markdown
| 知识点 | 难度 |
|--------|------|
| 遗传学 | 中等 |
```

#### 方法：`_markdown_table_to_html(markdown) → str`

**功能：** 将Markdown表格转换为HTML（用于前端渲染）

**输出示例：**
```html
<table class="word-table">
  <thead>
    <tr><th>知识点</th><th>难度</th></tr>
  </thead>
  <tbody>
    <tr><td>遗传学</td><td>中等</td></tr>
  </tbody>
</table>
```

### 4.2 Gemini API调用（OpenAI兼容模式）

**文件：** `backend/gemini_analyzer.py`

#### 初始化

### 4.3 前端渲染

**文件：** `frontend/src/components/ResultDisplay.jsx`

#### 函数：`renderStructuredContent(elements)`

**功能：** 按元素类型渲染内容

```jsx
const renderStructuredContent = (elements) => {
  return elements.map((element, idx) => {
    switch (element.type) {
      case 'paragraph':
        return <p>{element.content}</p>

      case 'table':
        return (
          <div dangerouslySetInnerHTML={{ __html: element.html }} />
        )

      case 'image':
        return (
          <img src={`data:image/jpeg;base64,${element.base64}`} />
        )
    }
  })
}
```

## 8. 已知问题与解决方案

### 8.1 已解决的问题

#### ✅ 问题1：Word文档表格无法渲染
**现象：** Word表格被提取为Markdown，前端无法正确显示
**根本原因：** 前端只支持HTML渲染，不支持Markdown表格
**解决方案：** 后端转换Markdown为HTML，前端使用`dangerouslySetInnerHTML`渲染

#### ✅ 问题2：Word图片丢失
**现象：** Word中的图片在题目中不显示
**根本原因：** 图片未被提取，或未传递给前端
**解决方案：** 提取图片→base64编码→通过`structured_content`传递→前端按顺序渲染

#### ✅ 问题3：Gemini API配额限制
**现象：** `429 You exceeded your current quota`
**根本原因：** Google免费API每天限制50次请求
**解决方案：** 切换到第三方服务商（chataiapi.com），无速率限制

#### ✅ 问题4：JSON解析失败（控制字符）
**现象：** `Invalid control character at: line 8 column 1181`
**根本原因：** Gemini返回的JSON包含非法ASCII控制字符
**解决方案：** `extract_json()` 方法自动清理控制字符

#### ✅ 问题5：跨页题目被拆分
**现象：** 题目14、16的题干和选项被分成两个题目
**根本原因：** Prompt未明确说明跨页题目处理规则
**解决方案：** 增强Prompt，明确"题号是唯一标识，相同题号属于同一题"

### 8.2 潜在风险

#### ⚠️ 风险1：第三方API服务稳定性
**影响：** API服务商可能不稳定或关闭
**缓解措施：**
- 保留Google官方API配置（注释状态）
- 可随时切换回官方API（需注意速率限制）
- 建议申请多个第三方API密钥作为备份

#### ⚠️ 风险2：大文件处理超时
**影响：** 20页以上的试卷可能处理超时
**缓解措施：**
- 增加 `timeout` 配置（当前60秒）
- 优化图片压缩质量（当前85）
- 考虑分批处理大文件

#### ⚠️ 风险3：内存溢出
**影响：** 处理多个大文件时可能OOM
**缓解措施：**
- 当前限制 `memory: 1400M`
- 及时清理临时文件
- 建议升级服务器配置至2核4G
